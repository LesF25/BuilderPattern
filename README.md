# Демонстрационный пример работы паттерна Builder.

Builder - паттерн проектирования, крторый позволяет создавать сложные объекты пошагово.
Он позволяет использовать один и тот же код для получения разных представлений объектов.

## Применение паттерна.

### Проблематика:

Cоздание конструктора с большим количеством параметров, которые большую часть
времени не будут использоваться, а вызов самого конструктора будет выглядеть громоздко.

### Решение проблемы:

Наш паттерны предлагает вынести конструирование объекта за пределы его собственного класса,
поручив это строителям. Паттерн предлагает разбить процесс конструирования объекта на отдельные шаги
(методы, которые мы будем вызывать для производства нужного нам вида объекта).

Директор - будет задавать порядок шагов строительства, а строители - выполнять их.
В этом случае нам останется только привязать делаемого строителя к директору,
а затем получить у строителя готовый результат.

### Предметная область:

Построение отчётов об игроках баскетбольной команды. Директор будет указывать конкретную информацию, которая ему может понадобиться в отчёте.
Строитель будет формировать отчёт и возвращать его.

В дальнейшем планируется подобрать более удачный пример его реализации
(универсальное построение различного рода отчётов, к примеру, об игроках/рабочих и т.д.).
